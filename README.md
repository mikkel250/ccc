# CV Tailoring API

API-only Next.js 15 backend: `POST /api/tailor-cv` accepts a job description and returns a tailored CV as a base64-encoded `.docx`, generated by an LLM against a private career knowledge base.

## Overview

- **What it does:** Compiles all career knowledge-base files, calls an LLM to tailor the CV to the supplied job description, and returns a Word document (base64 `.docx`).
- **Who calls it:** The CCC consumer app, which decodes the `.docx` and attaches it to a Gmail draft.
- **No frontend, no auth, no database** (MVP — stateless).

---

## Prerequisites

- Node.js ≥ 22.0.0 (`node --version` to verify; `.nvmrc` present for nvm users)
- npm (bundled with Node)
- At least one LLM API key: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`, `OPENROUTER_API_KEY`, or `DEEPSEEK_API_KEY`
- `TAILOR_MODEL` set to a namespaced `provider/model` string (e.g. `anthropic/sonnet`)

---

## Local Setup

```bash
# 1. Copy env template
cp .env.example .env.local

# 2. Fill in at least one LLM API key and set TAILOR_MODEL
#    (open .env.local in your editor)

# 3. Install dependencies
npm install

# 4. Start the dev server
npm run dev
# → http://localhost:3000

# 5. Verify it's running
curl http://localhost:3000/api/hello
```

> **E2E env note:** The manual e2e script (`scripts/e2e-tailor-cv.ts`) reads `.env` by default, not `.env.local`. For e2e runs: `cp .env.local .env`.

## Environment Variables

Brief summary of key variables. Full catalog with comments: [`.env.example`](.env.example)

| Variable | Purpose | Default |
|----------|---------|---------|
| `TAILOR_MODEL` | LLM used for CV generation (`provider/model`) | `anthropic/sonnet` |
| `OPENROUTER_API_KEY` | OpenRouter unified gateway (enables all `openrouter/*` models) | — |
| `ANTHROPIC_API_KEY` | Direct Anthropic access | — |
| `GOOGLE_API_KEY` | Direct Google Gemini access | — |
| `OPENAI_API_KEY` | Direct OpenAI access | — |
| `DEEPSEEK_API_KEY` | Direct DeepSeek access | — |
| `RATE_LIMIT_MAX` | Max requests per IP per window | `5` |
| `RATE_LIMIT_WINDOW` | Window duration in ms | `60000` |
| `LANGFUSE_TRACING` | Enable Langfuse tracing (`true`/`false`) | `false` |

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/hello` | Health probe — confirms the process is up |
| `POST` | `/api/tailor-cv` | Accepts `{ jobDescription }`, returns `{ cv: base64docx, model, usage, remaining, resetTime }` |

Full request/response shapes, all status codes, and error envelope: [`docs/api/API.md`](docs/api/API.md)

## Quick Examples

```bash
# Health check
curl http://localhost:3000/api/hello

# Tailor a CV (save to docx)
curl -s -X POST http://localhost:3000/api/tailor-cv \
  -H "Content-Type: application/json" \
  -d '{"jobDescription": "Senior React engineer. Requirements: TypeScript, React, Node."}' \
  | jq -r '.cv' | base64 -d > /tmp/cv.docx
```

## Testing

### Unit tests (no server, no API keys required)

```bash
npm test
```

### E2E tests (requires running dev server + API keys)

```bash
npm run dev          # in one terminal
npm run test:e2e     # in another
```

Set `RUN_E2E_LLM_TESTS=true` to include the full LLM call test (costs tokens).

### Manual multi-JD e2e script

```bash
# Run all 3 sample job descriptions
npx tsx scripts/e2e-tailor-cv.ts http://localhost:3000

# Run one sample at a time (for free-tier providers)
npx tsx scripts/e2e-tailor-cv.ts http://localhost:3000 ai-ml
```

See [`docs/workingDocs/TESTING.md`](docs/workingDocs/TESTING.md) for the full test reference (unit test table, rate-limit troubleshooting, Langfuse trace setup, Gemini free-tier workarounds).
